## 1. 개요 (Overview)

### 1.1 제품 비전
인천공항 내 **공항 내비게이션** 및 **항공 퀴즈 서비스**를 제공하는 모듈.
Telegram 봇을 통한 항공 퀴즈 자동 발송, 공항 내 경로 탐색, 게이트/편의시설 안내 기능을 통합 제공한다.

### 1.2 문제 정의

| # | 문제 | 현재 해결 방식 | 한계 |
|---|------|--------------|------|
| 1 | 공항 내 위치 파악 어려움 | 안내 표지판, 직원 문의 | 실시간 경로 탐색 불가 |
| 2 | 항공 관련 지식 습득 채널 부재 | 교재/유튜브 | 일방향 콘텐츠, 인터랙션 없음 |
| 3 | 편의시설 위치 정보 파편화 | 앱마다 다름 | 통합 조회 불가 |

### 1.3 목표 (Goals)

- **Primary**: Telegram 봇을 통한 항공 퀴즈 자동화 제공 (하루 3회)
- **Secondary**: 공항 내 웨이포인트 기반 경로 탐색 (Dijkstra 알고리즘)
- **Non-goal**: 실시간 항공편 예약, 체크인 연동 (외부 시스템)

## 3. 핵심 기능 (Core Features)

### 3.1 구현된 기능

| 우선순위 | 기능명 | 설명 | 대상 |
|---------|--------|------|------|
| 🔴 P0 | **항공 퀴즈 봇** | Telegram 봇으로 하루 3회(09:00, 14:00, 20:00 KST) 퀴즈 자동 발송 | 구독자 |
| 🔴 P0 | **Telegram 구독 관리** | 사용자 구독/해지, 언어 설정, 활동 추적 | 전체 |
| 🔴 P0 | **공항 내비게이션** | 터미널·층·웨이포인트 기반 경로 탐색 (Dijkstra) | 이용객 |
| 🟡 P1 | **웨이포인트 관리** | 게이트·카운터·화장실·라운지 등 POI 좌표 관리 | 관리자 |
| 🟡 P1 | **비행 게이트 추적** | 항공편별 게이트 현황 (탑승 상태) 조회 | 이용객 |
| 🟡 P1 | **AI 퀴즈 생성** | Anthropic / Gemini / Ollama 기반 자동 문제 생성 | 시스템 |
| 🟡 P1 | **사용자 위치 추적** | GPS/수동/WiFi/비콘 기반 위치 기록 | 시스템 |
| 🟢 P2 | **관리자 대시보드** | 퀴즈 토픽, 날씨, 백업 관리 웹 UI | 관리자 |

### 3.2 AI 퀴즈 프로바이더

| 프로바이더 | 용도 |
|-----------|------|
| Anthropic Claude | 주 퀴즈 생성 |
| Google Gemini | 대체 생성 |
| Ollama | 로컬 개발/테스트 |
| Google AI Studio | 추가 실험 |

## 4. 기술 요구사항 (Technical Requirements)

### 4.1 시스템 구성

```
Backend   : Node.js / TypeScript (독립 서비스, 포트 3010)
Database  : MySQL (Prisma ORM)
Bot       : Telegram Bot API (polling 방식)
AI        : Anthropic API, Google Gemini API, Ollama (로컬)
Frontend  : Vue 3 + Quasar (관리자 대시보드)
```

### 4.2 비기능 요구사항

| 항목 | 요구 수준 |
|------|----------|
| 퀴즈 발송 시간 | ±5분 허용 (cron 기반) |
| 경로 탐색 응답 | 500ms 이하 |
| Telegram 폴링 | 장애 시 자동 재시작 |

## 5. 데이터 모델 (Core Data Model)

```
NavTerminal (터미널)
  └── NavFloor (층)
        └── NavWaypoint (웨이포인트 / POI)
              └── NavWaypointConnection (연결 경로)
              └── NavFlightGate (게이트 배정)

NavUserLocation (사용자 위치)
NavNavigationRoute (경로 탐색 결과)

TelegramUser (구독자)
```
